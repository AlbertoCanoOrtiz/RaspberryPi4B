# Raspberry Pi Kubernetes (K8s) Cluster Setup

This section outlines the steps to configure a lightweight Kubernetes cluster using Raspberry Pi nodes.

## 1. Master Node (Control Plane) Configuration

Run these commands on your primary node to initialize the cluster:
Bash

### 1. Prepare repositories

```Bash

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | \
sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-archive-keyring.gpg
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-archive-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /" | \
sudo tee /etc/apt/sources.list.d/kubernetes.list
```

### 2. Install Kubernetes components

```Bash

sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo systemctl enable --now kubelet
```

### 3. Initialize cluster
#### Replace IP with your Master Node's static IP

```Bash

sudo kubeadm init --apiserver-advertise-address=xxx.xxx.xxx.xxx --pod-network-cidr=10.244.0.0/16
```

### 4. Configure kubectl for admin

```Bash

mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

### 5. Install Flannel CNI (Network Overlay)

```Bash

kubectl apply -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml
```

## 2. Worker Node(s) Setup

After the Master is ready, run these on each worker node:
Bash

### 1. Prepare repositories

```Bash

sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.29/deb/Release.key | \
sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-archive-keyring.gpg
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-archive-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.29/deb/ /" | \
sudo tee /etc/apt/sources.list.d/kubernetes.list
```

### 2. Install Kubernetes components

```Bash

sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo systemctl enable --now kubelet
```

### 2. Ensure container runtime is running

```Bash

sudo systemctl restart containerd
sudo systemctl status containerd
```

### 3. Join the cluster
#### Use the token and hash generated by the 'kubeadm init' command on the Master

```Bash

sudo kubeadm join xxx.xxx.xxx.xxx:6443 --token <TOKEN> \
  --discovery-token-ca-cert-hash sha256:<HASH>
```

## 3. Remote Client Access

To manage the cluster from your laptop or local workstation:
Bash

### 1. Install kubectl on your client machine

### 2. Copy kubeconfig from Master

```Bash

mkdir -p ~/.kube
scp user@xxx.xxx.xxx:/etc/kubernetes/admin.conf ~/.kube/config
chmod 600 ~/.kube/config
```

### 3. Test connectivity

```Bash

kubectl get nodes
```
